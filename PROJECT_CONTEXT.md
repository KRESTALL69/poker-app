# Project Context

## Что это

`Don't worry club` — это Telegram Mini App для покерных турниров.

Проект уже развернут в новом окружении:

- новый GitHub репозиторий
- новый Vercel project
- новый Supabase project
- новый Telegram bot
- все env-переменные уже обновлены

Приложение уже рабочее: деплой есть, база подключена, схема перенесена, Telegram Mini App открывается корректно.

## Архитектура

- Next.js App Router
- TypeScript
- Tailwind CSS
- Supabase
- Telegram Mini App / Telegram Bot
- Vercel

## Основные сущности

- `players` — игроки, профиль, роль, Telegram-данные, ник
- `tournaments` — турниры, дата, лимит игроков, описание, локация, статус
- `registrations` — записи на турниры, waitlist, отмены
- `results` — результаты турниров, место, нокауты, очки рейтинга
- `seasons` — сезоны
- `player_achievements` — прогресс достижений игрока

## Где что лежит

- `app/` — страницы и UI на App Router
- `app/api/` — API routes и webhook routes
- `features/` — бизнес-логика и работа с Supabase
- `lib/` — инфраструктурные helper-функции и интеграции
- `types/` — domain/database types
- `components/` — переиспользуемые UI-компоненты

## Что уже реализовано

- главная страница и навигация внутри Mini App
- список турниров и детальная страница турнира
- регистрация, waitlist, отмена регистрации
- профиль игрока
- рейтинг и результаты турниров
- админская часть для работы с турнирами
- Telegram webhook для бота
- достижения и прогресс достижений
- server-side sync достижений после сохранения результатов турнира

## Важные правила работы

- не пересобирать архитектуру без необходимости
- вносить точечные изменения
- не ломать существующую логику, маршруты и интеграции
- переиспользовать текущие `features/`, `lib/`, `types/`
- аккуратно работать с русским текстом и UTF-8
- если задача про UI, не трогать бизнес-логику без необходимости

## Как лучше работать в новых task

Перед изменениями:

1. короткий анализ
2. краткий план
3. список файлов
4. preview diff
5. только после подтверждения применять правки

## Дополнительно

- проект — это копия уже существующего покерного приложения, но в новом окружении
- Supabase пустой только с точки зрения данных, но схема уже перенесена
- если появляются новые сущности, сначала лучше проверить, можно ли встроить их в текущую архитектуру без лишнего усложнения
